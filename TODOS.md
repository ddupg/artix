# TODOS

## Homebrew 分发

What:
为 `artix` 增加 Homebrew tap 分发。

Why:
v1 已经决定包含 GitHub Releases，但 Homebrew 被延后。对 CLI/TUI 工具来说，安装摩擦会直接影响实际使用率，这不是纯包装工作。

Pros:
- 降低 macOS 用户安装门槛
- 对外分享时更容易推广和复用
- 让分发路径更完整，产品感更强

Cons:
- 会扩大发布面和维护面
- 需要维护 tap 配置和版本发布配套动作

Context:
当前评审已经把 GitHub Releases 纳入 v1 工程边界，所以 Homebrew 不是“要不要分发”，而是“分发还要不要再走一步”。这件事不阻塞先把产品做对，但值得明确记录，避免以后发布时临时想起来。

Depends on / blocked by:
- 依赖 GitHub Releases 先稳定
- 更适合在 v1 可用、发布流程跑通后跟进

## 扫描中增量刷新 UI

What:
支持扫描过程中增量刷新总览和详情，而不是等扫描完成后一次性展示。

Why:
本轮性能评审里，v1 有意识放弃了增量刷新，换取更低状态复杂度和更稳的 UI。这没问题，但它是明确的二期体验升级点，不应该无声消失。

Pros:
- 打开工具后反馈更快，体感更接近 `ncdu`
- 用户能更早知道哪些项目最肥
- 更适合大工作区、慢磁盘和长扫描场景

Cons:
- 会显著提高 UI 状态复杂度
- 会重新引入排序抖动、勾选状态保持、局部刷新一致性等测试负担

Context:
这不是“没做完”，而是本轮评审中主动放弃的复杂度。把它写成 TODO 的意义，是以后有人追求更爽体验时，知道这是一个清晰的二期方向，而不是临时起意乱改。

Depends on / blocked by:
- 依赖扫描流水线和 UI 状态先稳定
- 更适合在 v1 扫描与删除主链路跑顺后再做
